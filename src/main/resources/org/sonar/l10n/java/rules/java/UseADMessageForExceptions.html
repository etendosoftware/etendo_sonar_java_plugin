Exception messages should never be hard-coded. An AD_MESSAGE entry should be created for each message to be thrown. This will allow the possibility to translate the messages to other languages, apart from it being easier to modify if needed.

Noncompliant Code Example

    public void someMethod() {
      // ...
      try {
        // ...
      } catch (Exception e) {
        throw new IllegalArgumentException("Hard-coded message"); // Noncompliant
      }
    }

Compliant Solution

    public void someMethod() {
      // ...
      try {
        // ...
      } catch (Exception e) {
        throw new IllegalArgumentException(OBMessageUtils.messageBD("AD_MESSAGE_KEY")); // Compliant
      }
    }
