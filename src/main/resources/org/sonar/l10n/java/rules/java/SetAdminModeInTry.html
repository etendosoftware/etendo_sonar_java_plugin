The OBContext.setAdminMode() method should always be called inside a try-catch block to handle any errors that could come from it, as this method failing would leave the software in an unstable state

Noncompliant Code Example

  public void doSomethingMethod() {
    // ...
    OBContext.setAdminMode(true); // Noncompliant
    try {
      // ...
    } catch (Exception e) {
      // ...
    }
  }

Compliant Solution

  public void doSomethingMethod() {
    // ...
    try {
      OBContext.setAdminMode(true); // Compliant
      // ...
    } catch (Exception e) {
      // ...
    }
  }
