If the OBContext.setAdminMode() method is being called inside a try-catch block, then there has to exist a finally block for it, inside which the OBContext.restorePreviousMode() method must be called. This will prevent problems with the OBContext for other methods

Noncompliant Code Example

  try {
    OBContext.setAdminMode(true); // Noncompliant
  } catch (Exception e) {
    throw new Exception("This is an exception");
  }

Compliant Solution

  try {
    OBContext.setAdminMode(true); // Compliant
  } catch (Exception e) {
    throw new Exception("This is an exception");
  } finally {
    OBContext.restorePreviousMode();
  }
